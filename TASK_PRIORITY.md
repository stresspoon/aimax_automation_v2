#### 2. 고객모집 자동화 실제 연동 (Step 1~3)

Step 1: 고객모집 글쓰기 (Gemini Only)
- [ ] UI: 탭 전환 (블로그/스레드), 입력필드(본인 API 키, 네이버 블로그 ID, 작성 지침)
- [ ] 지침 템플릿 + "지침 수정 가이드" 툴팁
- [ ] 텍스트 생성(Gemini 2.5 Pro)
- [ ] 이미지 생성(Gemini Image) — 결과 복사/저장
- [ ] 결과 복사(텍스트/이미지), 프로젝트 카드에 결과 스냅샷 저장

Step 2: DB 관리 (시트 연동 + 자동 선별)
- [ ] 상단 노션 매뉴얼 링크(구글폼+구글시트 셋업)
- [ ] 시트 링크 입력 → 시트 읽기 (MVP: 공개된 시트의 CSV/Sheets API Key 기반 읽기)
- [ ] 러너(일시정지/재개): 마지막 동기화 시각 저장, 증분 동기화
- [ ] 후보 레코드 스키마(Candidate) 저장: 이름/이메일/전화/채널링크/측정값/분류
- [ ] 자동 선별 기준 적용(기본값: Threads≥500, Naver Blog≥300, Instagram≥1000)
- [ ] 기준값 UI(고정값으로 시작 → 추후 수정 가능 UI)
- [ ] 선정/비선정으로 파티셔닝 → 프로젝트에 바인딩
- [ ] (선택) 외부 크롤러/데이터 API(예: Apify) 연동 트랙: 공식 API 부재 채널 대응

Step 3: 이메일 발송
- [x] (임시) 이메일 발송 API/템플릿(SendGrid) — Gmail OAuth로 전환 예정
- [ ] Gmail OAuth 연동(사용자 계정 인증) — 발신자 Gmail만 허용
- [ ] 템플릿 관리(개인화 {이름} 플레이스홀더)
- [ ] 본문 자동작성(Gemini 2.5 Pro) + 지침 템플릿/가이드
- [ ] 선정/비선정 대상 토글, 개별 발송, 발송 로그 저장
- [ ] 레이트 리밋/쿼터 가드

프로젝트 카드 & 과금
- [ ] 프로젝트 카드 상태(S1/S2/S3), 생성글/시트링크/선정DB/템플릿 이력 묶음 저장
- [ ] Step 1 무료 3회: 사용자별 카운터/차단 + 구매 유도 모달(현재는 외부 결제 링크)

대시보드 IA (메인 진입 구조)
- [ ] 최상단 "자동화" 섹션: 고객모집 자동화(진입), 상세페이지/영상 자동화(준비중 표시)
- [ ] 하단 "개별 판매 도구" 섹션: 6개 카드(모두 구매 진입 가능)

개별 판매 도구(6종)
- [ ] 카드 UI 및 장바구니 담기 액션
- [ ] 상세 설명 페이지(간단) → 장바구니/외부 결제 링크 연결

장바구니/구매
- [ ] 로컬 장바구니 담기/삭제/수량조절
- [ ] "결제하기" → 임시 외부 플랫폼 딥링크(향후 PG 스위칭)

#### 3. 사용자 대시보드 데이터 연동
